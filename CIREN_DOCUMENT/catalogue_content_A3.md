### ② 「CIRENとは」— 概要説明（2〜3行）

```
CIREN は、センサーノードを接続するだけで稼働する
産業用 IoT モニタリングシステムです。
温度・湿度・振動・電圧など多種センサーに対応し、
取得データはリアルタイムでクラウドダッシュボードに表示されます。
WiFi・LTE-M（Cat-M1）デュアル接続により、
工場・屋外・遠隔地を問わず安定した監視環境を実現します。
```

---

### ③ 3つの価値提案（アイコン付き推奨）

| # | タイトル | 説明 |
|---|----------|------|
| 1 | **導入がシンプル** | センサーを差し込むだけで自動認識。複雑な設定・配線工事は不要。 |
| 2 | **マルチセンサー対応** | 温度・湿度・振動（IMU）・電圧など、用途に応じてセンサーを自由に組み合わせ可能。 |
| 3 | **つながり続ける設計** | WiFi 圏外でも LTE-M（Cat-M1）に自動切替。データ欠損ゼロを目指した冗長構成。 |

### ⑥ システム仕様

#### ハードウェア構成

| コンポーネント | モデル / 仕様 | 役割 |
|----------------|--------------|------|
| メインモジュール MCU | ESP32-S3 | データ集約・WiFi/MQTT 送信 |
| モバイル通信 | SIM7080G (Cat-M1 / NB-IoT) | LTE-M フォールバック接続 |
| ディスプレイ | TFT ILI9341 2.4" (320×240) | 現地ステータス表示 |
| コントローラー MCU | ESP32 | センサーノード集約（最大8台） |
| センサーノード MCU | Seeeduino XIAO | センサーインターフェース |
| 無線プロトコル | ESP-NOW（チャンネル固定） | コントローラー↔メインモジュール間 |

#### ソフトウェア・通信

| 項目 | 仕様 |
|------|------|
| RTOS | FreeRTOS（マルチタスク、デュアルコア） |
| MQTT ブローカー | Mosquitto |
| バックエンド | Node.js + Express + MongoDB |
| フロントエンド | React 19 + Vite + Tailwind CSS |
| データ通信 | MQTT (QoS 0) + WebSocket（リアルタイム） |
| 接続モード | WiFi（優先）/ LTE-M Cat-M1（自動フォールバック） |
| OTA 対応 | ファームウェアバージョン管理対応 |

---

### ⑦ 対応センサー一覧

| センサー | 計測項目 | ノード型番 |
|----------|----------|-----------|
| DHT20 (I2C) | 温度 / 湿度 | node_dht20 |
| MPU6050 (I2C) | 加速度 (X/Y/Z)・ジャイロ (X/Y/Z)・姿勢角 (Pitch/Roll/Yaw) | node_mpu6050 |
| 電圧センサー | 電圧 (0〜5V) | node_voltage |
| *(拡張可能)* | カスタムセンサー対応（シリアル通信経由） | — |

---

### ⑧ 主な機能

```
✔ センサーのホットスワップ対応（電源オフ不要で交換・追加可能）
✔ ノードごとのデータ送信間隔をダッシュボードから設定可能
✔ データ履歴の閲覧・CSV/JSON エクスポート
✔ アラート閾値設定（センサー値が範囲を超えた際に通知）
✔ GPS 位置情報表示（Leaflet マップ連携）
✔ ハードウェアウォッチドッグによる自動復旧
✔ 複数デバイス・複数拠点の一元管理
```
